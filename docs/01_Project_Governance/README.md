# 01 Project Governance

## Purpose

Defines why the project exists and the rules under which it operates: constitution, vision, operating principles, roadmap, and AI instructions.

## Contents

Constitution, Project Vision, Core Philosophy (the operating principles), Project Roadmap, Claude Instructions, Project Rules pointer. Filenames retain their original numeric prefixes until the deliberate rename commit.

## Relationships

Authoritative over every other folder. PROJECT_RULES.md at the repository root remains the binding rules document; this folder governs everything else.

## How This Folder Should Evolve

Changes rarely and deliberately. Every change is versioned and dated. Nothing here is deleted; superseded principles are archived.
